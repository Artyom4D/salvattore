How it works?

Simply add a data-columns attribute to the element that contains the blocks you want on your layout. All the settings and conditions are defined in CSS.

<div id="timeline" data-columns>
  <div>Item #1</div>
  <div>Item #2</div>
  <div>Item #3</div>
  <div>Item #4</div>
  <div>Item #5</div>
</div>

In your CSS file you can set the number of columns you want to be created inside that element. You can even use media queries.

#timeline[data-columns] {
    content: '2';
}

@media screen and (min-width: 601px){
  #timeline[data-columns] {
    content: '3';
  }
}

Use the content property as a wildcard to tell Salvattore how many columns you want. The script will create columns as elements where each of the original element’s items will be distributed mathematically in these new columns.

<div id="timeline" data-columns>
  <div class="column">
    <div>Item #001</div>
    <div>Item #004</div>
  </div>
  <div class="column">
    <div>Item #002</div>
    <div>Item #005</div>
  </div>
  <div class="column">
    <div>Item #003</div>
  </div>
</div>

Customization

The class for each column is .column by default, but you can also re-define it or even use more than one through CSS so that you have a flexible and modular code.

#timeline[data-columns] {
  content: '2 .col.one-half';
}

@media screen and (min-width: 601px){
  #timeline[data-columns] {
    content: '3 .col.one-third';
  }
}

Adding dynamic content

Salvattore has two functions you can work with: append_elementsand prepend_elements. These two functions have the same attributes: the gridand the elementsyou want added to it. So for instance: 

var grid = document.querySelector('#timeline');
var item = document.createElement('article');

salvattore['append_elements'](grid, [item])
item.outerHTML = 'I’ve been appended!';

Prepend content Append content

Items are appended and prepended on the same grid without conflicts. You can combine this API with your XHR functions to have a dynamic layout with live content.

Compatibility

Salvattore works in Safari, Chrome, Firefox and IE9+. Feel free to help us make it work for other browsers.
